Thu thập log từ WAF

3.1 Xác định nơi lưu log và đảm bảo dữ liệu đã có

Mục tiêu: Đảm bảo WAF đang ghi log, và log được lưu ở một nơi có thể truy cập để xử lý.

Việc cần làm:

  1. Vào AWS WAF → WebACL → Logging để kiểm tra đã bật logging.
  2. Xem log đang lưu ở đâu:
    • CloudWatch Logs: Tạo Resource Policy cho phép WAF ghi log.
    • Amazon S3: Tạo bucket và cấp quyền cho WAF.
  3. Mở CloudWatch Logs hoặc S3 để chắc chắn log có request thực tế.

📌 Lưu ý: Nếu log trống, hãy gửi vài request vào CloudFront hoặc API Gateway mà WebACL đang bảo vệ.

3.2 Thu thập dữ liệu log

Mục tiêu: Lấy dữ liệu log ra file hoặc vào chương trình Python để xử lý.

  • Nếu log ở CloudWatch Logs:

    aws logs filter-log-events       --log-group-name "Tên-log-group"       --region us-east-1       --limit 50

  • Nếu log ở S3:

    aws s3 cp s3://your-bucket-name/AWSLogs/... ./logs.json

3.3 Tiền xử lý & trích xuất đặc trưng (Feature Engineering)

Mục tiêu: Từ log JSON, lấy ra thông tin cần cho ML.

  • Trích xuất các trường quan trọng:

    • clientIp
    • country
    • httpMethod
    • uri
    • requestSize
    • action (ALLOW / BLOCK / COUNT)
    • labels

  • Ví dụ với Pandas:

    import pandas as pd

data = []
for record in logs['records']:
    req = record['httpRequest']
    data.append({
        'ip': req['clientIp'],
        'country': req.get('country'),
        'method': req['httpMethod'],
        'uri': req['uri'],
        'size': req['requestSize'],
        'action': record['action']
    })

df = pd.DataFrame(data)
print(df.head())

3.4 Áp dụng thuật toán ML để phát hiện bất thường

Mục tiêu: Tìm request bất thường so với hành vi bình thường.

  • Thuật toán đề xuất:

    • Isolation Forest
    • One-Class SVM
    • Local Outlier Factor (LOF)

  • Ví dụ với Isolation Forest:

    from sklearn.ensemble import IsolationForest

features = df[['size']]
model = IsolationForest(contamination=0.01, random_state=42)
df['anomaly'] = model.fit_predict(features)

anomalies = df[df['anomaly'] == -1]
print(anomalies)

3.5 Tự động cập nhật luật WAF

Mục tiêu: Chặn IP hoặc pattern bất thường từ kết quả ML.

  • Các bước:

    1. Lấy danh sách IP bất thường từ df.
    2. Tạo hoặc cập nhật IP set trong AWS WAF bằng AWS CLI hoặc boto3.
    3. Gắn IP set vào Rule trong WebACL (Action = BLOCK).

  • Ví dụ boto3:

    import boto3

waf = boto3.client('wafv2', region_name='us-east-1')
waf.update_ip_set(
    Name='MaliciousIPs',
    Scope='CLOUDFRONT',
    Id='IPSetId',
    Addresses=['192.168.1.1/32', '203.0.113.0/24'],
    LockToken='lock-token-here'
)
